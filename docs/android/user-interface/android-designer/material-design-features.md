# <a name="xamarinandroid-designer-material-design-features"></a>Функции конструктора материальному дизайну Xamarin.Android

_В этом разделе описывает функции конструктора, которые упрощают для разработчиков для создания макетов материала проектирования с. В этом разделе представлены и описываются способы использования сетки материал, материал цветовую палитру, типографических масштаб и редактор темы._

> [!Video https://youtube.com/embed/E3_ZjIOzVzY]

**Усовершенствовано в 2016 г: Все пользователи могут создавать красивые приложения с материального исполнения**

## <a name="overview"></a>Обзор

Конструктор Xamarin.Android включает в себя функции, которые упрощают создание макетов материал-конструктора совместимые. Если вы не знакомы с материальному дизайну, см. в разделе [Общие сведения о проектировании материал](https://material.io/design/introduction).

# <a name="visual-studiotabwindows"></a>[Visual Studio](#tab/windows)

В этом руководстве мы будем взглянем на следующие функции конструктора:

-   *Сетка материала* &ndash; наложение в области конструктора, показывающий сетки, интервалов и контуры, которые помогут поместить мини-приложениями макета в соответствии с рекомендациями по материальному дизайну.

-   *Редактор тем* &ndash; редактор ресурсов небольшого цвета, позволяет задать сведения о цвете для подмножества темы. Например, можно просмотреть и изменить цвета материала, такие как `colorPrimary`, `colorPrimaryDark`, и `colorAccent`.

Мы рассмотрим каждый из этих компонентов и приводятся примеры их использования.

## <a name="material-design-grid"></a>Таблица материального исполнения

Сетка конструктора материал меню доступно на панели инструментов в верхней части конструктора:

[![Материала сетку конструктора](material-design-features-images/vs/01-material-design-grid-w158-sml.png)](material-design-features-images/vs/01-material-design-grid-w158.png#lightbox)

Если щелкнуть значок бланк материал, конструктор отображает наложение в области конструктора, который включает в себя следующие элементы:

-   Контуры (оранжевый строки)

-   Интервал (зеленым)

-   Сетки (синие линии)

Эти элементы можно увидеть в предыдущем снимке экрана. Каждый из этих элементов наложения можно настроить. Когда вы нажмете кнопку с многоточием рядом с меню бланк материал, контекстном диалоговое окно открывается, позволяет отключить или включить сетки, настроить размещение контуры и задать расстояние между. Обратите внимание, что все значения представляются в `dp` (независимый от плотности пикселей):

[![Сетка, контуре и интервал конфигурации](material-design-features-images/vs/03-grid-configuration-w158-sml.png)](material-design-features-images/vs/03-grid-configuration-w158.png#lightbox)

Чтобы добавить новый рисунок, введите новое значение смещения в **смещение** окне выберите расположение (**левой**, **верхней**, **правой**, или  **нижней**) и нажмите кнопку + значок, чтобы добавить новый рисунок. Аналогичным образом, чтобы добавить новое значение интервала, введите размер и смещение (в dp) в **размер** и **смещение** флажков, соответственно. Выберите расположение (**левой**, **верхней**, **правой**, или **нижней**) и нажмите кнопку + значок, чтобы добавить новое значение интервала.

Если изменить эти значения конфигурации, они сохраняются в XML-файле разметки и используются всякий раз при повторном открытии макета.


## <a name="theme-editor"></a>Редактор темы

**Редактор тем** позволяет настраивать сведения о цвете для подмножества атрибутов темы. Чтобы открыть **редактор тем**, щелкните значок кисти на панели инструментов:

[![Значок темы редактора](material-design-features-images/vs/04-theme-editor-icon-w158-sml.png)](material-design-features-images/vs/04-theme-editor-icon-w158.png#lightbox)

Несмотря на то что **редактор тем** доступен на панели инструментов для всех целевых версии Android и уровнях API только определенный набор возможностей, описанных ниже доступны, если уровень целевой API более ранняя, чем API 21 (Android 5.0 Без описания операций).

На левой панели **редактор тем** отображает список цветов, образующий текущего выбранного темы (в этом примере мы используем `Default Theme`):

[![Редактор темы](material-design-features-images/vs/05-theme-editor-w158-sml.png)](material-design-features-images/vs/05-theme-editor-w158.png#lightbox)

При выборе цвета в левой части правую область содержит следующие вкладки для облегчения изменения цвета:

-   **Наследовать** &ndash; отображает схему наследования стиля для выбранного цвета, а также список разрешенных цвет и код цвета, назначенные цвета темы.

-   **Палитра цветов** &ndash; позволяет изменить на любое произвольное значение выбранного цвета.

-   **Палитра материалов** &ndash; позволяет изменить значение, которое соответствует материальному дизайну выбранного цвета.

-   **Ресурсы** &ndash; позволяет изменить выбранный цвет на один из других существующие ресурсы цвета в теме.

Давайте рассмотрим каждый из этих вкладках подробно.

### <a name="inherit-tab"></a>Наследовать вкладку

Как показано в следующем примере **наследовать** вкладке перечислены наследования стиля для **фона** цвет **темы по умолчанию**:

[![Наследовать вкладку](material-design-features-images/vs/06-inherit-tab-w158-sml.png)](material-design-features-images/vs/06-inherit-tab-w158.png#lightbox)

В этом примере **темы по умолчанию** наследует стиль, который использует `@color/background_material_light` , но переопределяет его с `color/material_grey_50`, которого имеет значение кода цвет `#fffafafa`.
Дополнительные сведения о наследовании стилей см. в разделе [стили и темы](http://developer.android.com/guide/topics/ui/themes.html#Inheritance).

### <a name="color-picker"></a>Палитра

На следующем снимке экрана показаны **палитра цветов**:

[![Палитра цветов](material-design-features-images/vs/07-color-picker-w158-sml.png)](material-design-features-images/vs/07-color-picker-w158.png#lightbox)

В этом примере **фона** цвета можно изменить на любое значение с помощью различных средств:

-   Если щелкнуть цвет напрямую.
-   Ввод значений цветового тона, насыщенности и яркости.
-   Ввод значения RGB (красный, зеленый, синий) в десятичном формате.
-   Настройка альфа-канала (прозрачности) для выбранного цвета.
-   Непосредственно введя шестнадцатеричного кода цвета.

Цвет в палитре цветов *не* ограниченные рекомендациям по материальному дизайну или набор доступных ресурсов.

### <a name="resources"></a>Ресурсы

**Ресурсы** вкладка содержит список цветовые ресурсы, которые уже присутствуют в теме:

[![Ресурсы](material-design-features-images/vs/08-resources-w158-sml.png)](material-design-features-images/vs/08-resources-w158.png#lightbox)

С помощью **ресурсы** вкладке ограничивает выбор в этот список цветов. Имейте в виду, что при необходимости ресурс цвета, который уже назначен другой частью темы, два соседних элемента пользовательского интерфейса может «запустить вместе» (поскольку они имеют тот же цвет) и будет затруднено для пользователя различать.

### <a name="material-palette"></a>Палитра материалов

**Палитры материал** откроется вкладка **материал разработки цветовую палитру**. Выбрав значение цвета из палитры ограничивает выбранный цвет, чтобы он был совместим с рекомендации по проектированию материалов:

[![Палитра материалов](material-design-features-images/vs/09-material-palette-w158-sml.png)](material-design-features-images/vs/09-material-palette-w158.png#lightbox)

Верхней части палитры отображаются основного цвета материальному дизайну, а диапазон тона для выбранной основной цвет нижней части палитры. Например, при выборе **Indigo**, коллекция **Indigo** тона отображается в нижней части диалогового окна.
При выборе оттенок цвета свойства меняется на выбранной цветовой тон. В следующем примере `Background Tint` кнопки меняется на *Indigo 500*:

![Выберите Indigo 500](material-design-features-images/vs/10-indigo-w158.png)

`Background Tint` имеет значение код цвета для *Indigo 500* (`#ff3f51b5`), и конструктор обновляет цвет фона, чтобы отразить это изменение:

[![Изменить оттенок фона](material-design-features-images/vs/11-background-tint-w158-sml.png)](material-design-features-images/vs/11-background-tint-w158.png#lightbox)

Дополнительные сведения о цветовой палитре материальному дизайну, см. в разделе материальному дизайну [каталог палитры цветов](https://material.io/design/color/).

### <a name="creating-a-new-theme"></a>Создание новой темы

В следующем примере мы будем использовать палитру материал для создания новой пользовательской темы. Во-первых, мы изменим **фона** цвет *синий 900*:

![Изменение фона для синего 900](material-design-features-images/vs/12-change-background-to-blue-w158.png)

При изменении ресурса цвета, с сообщением, появится сообщение *текущей теме есть несохраненные изменения*:

[![Предупреждение несохраненные изменения](material-design-features-images/vs/13-unsaved-changes-w158-sml.png)](material-design-features-images/vs/13-unsaved-changes-w158.png#lightbox)

**Фона** изменился цвет в конструкторе, чтобы новый выбранный цвет, но это изменение еще не были сохранены. На этом этапе необходимо выполнить одно из следующих действий.

-   Нажмите кнопку **отменить изменения** отменить новый выбранный цвет (или варианты) и вернуться в исходное состояние темы. 

-   Нажмите клавишу <kbd>CTRL + S</kbd> сохранить изменения, внесенные в данный момент темы. 

В следующем примере <kbd>CTRL + S</kbd> была нажата, чтобы изменения были сохранены в **AppTheme**:

[![Изменения, сохраненные в AppTheme](material-design-features-images/vs/14-custom-theme-w158-sml.png)](material-design-features-images/vs/14-custom-theme-w158.png#lightbox)

[![Значок темы редактора](material-design-features-images/xs/12a-theme-editor-icon-sml.png)](material-design-features-images/xs/12a-theme-editor-icon.png#lightbox)

Несмотря на то что **редактор тем** доступен на панели инструментов для всех целевых версии Android и уровнях API только определенный набор возможностей, описанных ниже доступны, если уровень целевой API более ранняя, чем API 21 (Android 5.0 Без описания операций).

На левой панели **редактор тем** отображает список цветов, образующий текущего выбранного темы (в этом примере мы используем `Default Theme`):

[![Редактор темы](material-design-features-images/xs/12b-theme-editor-sml.png)](material-design-features-images/xs/12b-theme-editor.png#lightbox)

При выборе цвета в левой части правую область содержит следующие вкладки для облегчения изменения цвета:

-   **Наследовать** &ndash; отображает схему наследования стиля для выбранного цвета, а также список разрешенных цвет и код цвета, назначенные цвета темы.

-   **Палитра цветов** &ndash; позволяет изменить на любое произвольное значение выбранного цвета.

-   **Палитра материалов** &ndash; позволяет изменить значение, которое соответствует материальному дизайну выбранного цвета.

-   **Ресурсы** &ndash; позволяет изменить выбранный цвет на один из других существующие ресурсы цвета в теме.

Давайте рассмотрим каждый из этих вкладках подробно.

### <a name="inherit-tab"></a>Наследовать вкладку

Как показано в следующем примере **наследовать** вкладке перечислены наследования стиля для **фона** цвет **темы по умолчанию**:

[![Наследовать вкладку](material-design-features-images/xs/13-inherit-sml.png)](material-design-features-images/xs/13-inherit.png#lightbox)

В этом примере **темы по умолчанию** наследует стиль, который использует `@color/background_material_dark` , но переопределяет его с `color/material_grey_850`, которого имеет значение кода цвет `#ff303030`.
Дополнительные сведения о наследовании стилей см. в разделе [стили и темы](http://developer.android.com/guide/topics/ui/themes.html#Inheritance).

### <a name="color-picker"></a>Палитра

На следующем снимке экрана показаны **палитра цветов**:

[![Палитра цветов](material-design-features-images/xs/14-color-picker-sml.png)](material-design-features-images/xs/14-color-picker.png#lightbox)


В этом примере **фона** цвета можно изменить на любое значение с помощью различных средств:

-   Если щелкнуть цвет напрямую.
-   Ввод значений цветового тона, насыщенности и яркости.
-   Ввод значения RGB (красный, зеленый, синий) в десятичном формате.
-   Настройка альфа-канала (прозрачности) для выбранного цвета.
-   Непосредственно введя шестнадцатеричного кода цвета.

Цвет в палитре цветов *не* ограниченные рекомендациям по материальному дизайну или набор доступных ресурсов.

### <a name="resources"></a>Ресурсы

**Ресурсы** вкладка содержит список цветовые ресурсы, которые уже присутствуют в теме:

[![Ресурсы](material-design-features-images/xs/15-resources-sml.png)](material-design-features-images/xs/15-resources.png#lightbox)

С помощью **ресурсы** вкладке ограничивает выбор в этот список цветов. Имейте в виду, что при необходимости ресурс цвета, который уже назначен другой частью темы, два соседних элемента пользовательского интерфейса может «запустить вместе» (поскольку они имеют тот же цвет) и будет затруднено для пользователя различать.

### <a name="material-palette"></a>Палитра материалов

**Палитры материал** откроется вкладка **материал разработки цветовую палитру** описано [более ранних](#material_palette). Выбрав значение цвета из палитры ограничивает выбранный цвет, так как это соответствует рекомендациям по материальному дизайну.

[![Палитра материалов](material-design-features-images/xs/16-material-palette-sml.png)](material-design-features-images/xs/16-material-palette.png#lightbox)

### <a name="creating-a-new-theme"></a>Создание новой темы

В следующем примере мы будем использовать палитру материал для создания новой пользовательской темы. Во-первых, мы изменим **фона** цвет *синий 900*:

[![Изменение фона для синего 900](material-design-features-images/xs/17-change-background-to-blue-sml.png)](material-design-features-images/xs/17-change-background-to-blue.png#lightbox)

При изменении ресурса цвета, с сообщением, появится сообщение *текущей теме есть несохраненные изменения*:

[![Предупреждение несохраненные изменения](material-design-features-images/xs/18-unsaved-changes-sml.png)](material-design-features-images/xs/18-unsaved-changes.png#lightbox)

Внесено изменение цвета в конструкторе, но это изменение еще не были сохранены. На этом этапе необходимо выполнить одно из следующих действий.

-   Нажмите кнопку **отменить изменения** отменить новый выбранный цвет (или варианты) и вернуться в исходное состояние темы. 

-   Нажмите клавишу  **&#8984; + S** чтобы сохранить изменения в новой темы с именем **Custom**.

В этом разделе описываются функции материальному дизайну, доступные в конструкторе Xamarin.Android. Кроме того, вы узнали, как использовать редактор темы для создания нового пользовательские темы, которые соответствуют рекомендациям по материальному дизайну.
