# <a name="introduction-to-ios-8"></a>Введение в iOS 8

_С iOS 8 Apple предоставляет множество новых платформ и API-интерфейсы для по вкусу и удовлетворения запросов разработчиков. В этом руководстве мы представить эти новые API и узнайте, как можно улучшить iOS 8, разработчиков и пользователей._

iOS 7 визуально изменить всего операций ввода-вывода пользовательского интерфейса из какие пользователи и разработчики имели следует ожидать, справа от первого iPhone операционной системы. IOS 8 продолжается с этим, предоставляя множество платформ для разработчиков, который позволяет пользователям управлять почти любым аспектом свою жизнь, от своего iPhone. Например здравоохранения и могут анализироваться с *HealthKit*, появляется в декларации с биометрической проверки подлинности с помощью секретных кодов *LocalAuthentication*, *расширений приложения*открыть канал связи между сторонние приложения, и *HomeKit* позволяет превратить ваш дом в домашней будущего. 

Если о восхищение пользователей iOS 7, iOS 8 посвящена восхищение разработчикам целый ряд этих tasty новых средств. 

Это руководство описывает новые интерфейсы API для разработчиков Xamarin.iOS.  

Существуют также несколько интерфейсов API, которые стали нерекомендуемыми в iOS 8, подробно описаны в конце этого документа.

## <a name="requirements"></a>Требования

Ниже перечислены необходимые для создания приложений iOS 8 в Visual Studio для Mac:

- **Xcode 7 и iOS 8 или более поздней версии** — последние Xcode и API для iOS компании Apple необходимо установить и настроить на компьютере разработчика.
- **Visual Studio для Mac** — последнюю версию Visual Studio для Mac должны быть установлены и настроены на устройстве пользователя.
- **iOS 8 устройства или симулятора** — устройства iOS под управлением последней версии iOS 8 для тестирования.

## <a name="home-and-leisure"></a>Дом и Досуг

iOS 8 помогла твердо внедрить Apple и устройство iOS прямо в сердце домашней сети за счет использования HomeKit и HealthKit. В этом разделе мы рассмотрим работать как оба этих новых платформ и как они могут быть интегрированы в приложение Xamarin.iOS.

## <a name="homekit"></a>HomeKit

Управление на устройствах с помощью iPhone не нового приложения технологии; Многие продукты подключенных домашней можно управлять с помощью приложения iOS. Тем не менее HomeKit теперь принимает это, гарантируя повысив уровень общего протокола для устройств автоматизации дома и предоставлении определенных производителей, таких как iHome, Philips Honeywell общедоступные API-Интерфейсы. Пользователю, это означает, что они могут управлять почти любым аспектом дома без проблем из внутри одного приложения. Он не имеет отношения к их о том, что они используют лампочки Philips Hue или сигнал вложенности. Пользователи также можно объединить в цепочку множество процессов смарт-home друг с другом в «Сцены».

С HomeKit сторонние приложения и Siri можно обнаружить стандартные и добавить их к базе данных их личных домашних конфигурации, изменить и обрабатывают эти данные и взаимодействовать с "Стандартные" и службы для выполнения действия.

## <a name="healthkit"></a>HealthKit

HealthKit — это платформа, впервые появился в iOS 8, который предоставляет централизованное, скоординированное и безопасное хранилище данных для получения сведений, связанных с работоспособностью. Операционная система обеспечивает конфиденциальность и безопасность, информация о состоянии работоспособности и работоспособности приложения, панели мониторинга для пользователя. С разрешения пользователя приложения могут читать и записывать широкий набор сведений о работоспособности.

Дополнительные сведения о ее использовании в приложении Xamarin.iOS см [введение в HealthKit](~/ios/platform/healthkit.md) руководства.

## <a name="extending-iphone-functionality"></a>Расширение функциональности iPhone
С помощью iOS8 разработчикам предоставляется гораздо больший контроль над кто может использовать свои приложения и повысить возможности для более открытого обмена данными между приложения сторонних производителей. Функции, такие как расширения приложений и средство выбора документа откройте целый мир новых возможностей использования приложений в экосистеме компании Apple.

### <a name="app-extensions"></a>Расширения приложений
Расширения приложения, проще говоря, предназначены для приложения сторонних производителей для взаимодействия друг с другом. В целях соблюдения стандартов высокого уровня безопасности и целостности изолированного приложения для того, это взаимодействие не происходит напрямую между приложениями. Вместо этого он выполняется *расширение* в середине.

Первым этапом создания расширения приложения является определение точки правильное расширение — это важно в обеспечении поведение и доступность правильный API-интерфейсы. Чтобы создать расширение приложения в Visual Studio для Mac, добавьте его в существующее приложение путем добавления проекта к решению.

В **новый проект** диалоговом окне перейдите к **C#**  >  **iOS** > **единый API**  >  **Расширения**, как показано на следующем снимке экрана:

![](introduction-to-ios8-images/image2.png "Создание нового расширения")
 
Диалоговое окно Новый проект предоставляет семь новых шаблонов проектов для создания расширений приложения и рассматриваются ниже. Обратите внимание на то, что большинство расширений связаны с другими новые интерфейсы API в iOS, например средство выбора документа:

- **Действие** — это позволяет разработчикам создавать уникальные пользовательские действия кнопки позволяет пользователям выполняет определенные задачи
- **Пользовательских сочетаний** — это позволяет разработчикам добавлять в диапазоне от встроенных Apple клавиатуры, добавив собственные настраиваемой классификации. Популярные клавиатуры, Swype использует это для приведения их сочетания операций ввода-вывода.
- **Document Picker** — этот файл содержит контроллер представления средство выбора документа, который позволяет пользователям получить доступ к файлам за пределами приложения "песочницы".
- **Документирование поставщик средства выбора файлов** — это обеспечивает безопасное хранение файлов, используя средство выбора документа.
- **Редактирование фотографий** — этот расширяет фильтры и средства, уже предоставляемых компанией Apple в приложении фотографий, чтобы предоставить пользователям больше контроля и дополнительные параметры при изменении его фотографии редактирования.
- **Сегодня** – это дает приложению возможность отображения мини-приложения в разделе "сегодня" центра уведомлений.

Дополнительные сведения об использовании расширений приложений в Xamarin см. [введение расширения приложения](~/ios/platform/extensions.md) руководства.

### <a name="touch-id"></a>Touch ID

Touch ID появился в iOS 7 с точки зрения проверки подлинности пользователя — аналогичен секретный код. Тем не менее он был ограничен разблокировки устройства, используя App Store, с помощью iTunes, а также для проверки подлинности только к цепочке ключей iCloud 

Теперь существует два способа использовать Touch ID в качестве механизма проверки подлинности в приложениях iOS 8, с помощью API локальной проверки подлинности. Он в настоящее время не поддерживается для использования локальной проверки подлинности для проверки подлинности удаленно.

Во-первых она также помогает существующих служб цепочки ключей при помощи новых цепочки ключей списки управления доступом (ACL). Данные цепочки ключей можно разблокировать с успешной проверкой подлинности пользователей пальца.

Во-вторых LocalAuthentication предоставляет два метода проверки подлинности приложения локально. Разработчикам следует использовать `CanEvaluatePolicy` для определения того, если устройство будет принимать Touch ID, а затем `EvaluatePolicy` для запуска операции проверки подлинности.

Дополнительные сведения о Touch ID и узнайте, как интегрировать его в приложении Xamarin.iOS см [введение к TouchID](~/ios/platform/touchid.md) руководства.

### <a name="document-picker"></a>Средство выбора документа

Работает средство выбора документа с диском iCloud пользователей, чтобы разрешить пользователю открывать файлы, которые были созданы в другое приложение, импортировать и управлять ими и экспортировать их обратно. При этом создается интуитивно понятный рабочий процесс и поэтому гораздо более удобным для пользователей. Синхронизация iCloud дальнейшей делает этот шаг, любые изменения, внесенные в одном приложении также может быть отражать постоянно на всех устройствах.

Дополнительные сведения о средстве выбора документа более подробно и узнайте, как интегрировать его в приложении Xamarin.iOS см. по [введение в средство выбора документов](~/ios/platform/document-picker.md) руководства.

### <a name="handoff"></a>Перемещение вручную

Перемещение вручную, который является частью крупной функции обеспечения непрерывности, принимает шаг вперед к интеграции OS X и iOS. Это включает функцию "airdrop кросс платформенные", может принимать вызовы iPhone, SMS на iPad и Mac, а также улучшения в модем с помощью iPhone.

Переадресуемое работает с iOS 8 и Yosemite и требуется учетная запись iCloud вход в систему на различные устройства, вы хотите использовать. Он должен работать с наиболее предварительно установленных приложений Apple, включая Safari, iWork, карты, календари и контакты.

Дополнительные сведения см. в разделе наших [Эстафетной](~/ios/platform/handoff.md) руководства.

## <a name="unified-storyboards"></a>Унифицированная раскадровка
iOS 8 входит новый проще в использовании механизм для создания пользовательского интерфейса — унифицированной раскадровки. С помощью одного раскадровки, чтобы охватить все размеры экрана другое оборудование быстрый и гибкий представления могут создаваться в стиле true «Проектируйте один раз, использовать многие».

До iOS8, разработчики использовали `UIInterfaceOrientation` для различения между режимами в альбомной и книжной ориентации и `UIInterfaceIdiom` провести различие между устройствами iOS. В iOS8 он больше не требуется создавать отдельный раскадровки для устройств iPhone и iPad, ориентацию и устройства определяются с помощью *классы размера*.

Все устройства определяется класс размер, в вертикальной и горизонтальной оси, и существует два типа классов размер в iOS 8:

- **Регулярные** -это мини-приложения, который создает впечатление большого размера (например, UIScrollView или размер большого экрана (таких как iPad)
- **Compact** -это для небольших устройств (таких как iPhone). Этот размер учитывает ориентации устройства.

Если эти два понятия используются совместно, результат — это сетка 2 x 2, определяет различные возможные размеры, которые могут использоваться в обоих разные ориентации, как показано на следующей схеме:

![](introduction-to-ios8-images/image3.png "Схемы, представляющей сетка 2 x 2, которая определяет различные возможные размеры, которые могут использоваться в разных ориентаций")
 
Дополнительные сведения о классах размер называть [Знакомство с унифицированным раскадровкам](~/ios/user-interface/storyboards/unified-storyboards.md).

## <a name="photo-kit"></a>Комплект фото
Комплект фото — это новая платформа, которая позволяет приложениям запрашивать библиотека изображений системы и создать настраиваемые пользовательские интерфейсы для просмотра и изменения его содержимого. Она включает множество классов, представляющих изображения и видео, а также коллекции средств, таких как диски и папки.

Дополнительные сведения см. в разделе наших [PhotoKit](~/ios/platform/photokit.md) руководства.

## <a name="games"></a>Игры

<a name="scenekit" />

### <a name="scene-kit"></a>Scene Kit

Scene Kit — это граф сцены API, который упрощает работу с трехмерной графикой. Он впервые появилась в OS X 10.8 и теперь пришло iOS 8. С помощью комплекта сцены создавать впечатляющие трехмерные визуализации и случайного трехмерных игр не требуется опыт в OpenGL. Основываясь на основные понятия, связанные с граф сцены, Scene Kit упрощает OpenGL и OpenGL ES, упрощая добавление трехмерного содержимого в приложение. Тем не менее если вы являетесь экспертом OpenGL, Scene Kit имеет хорошую поддержку привязки непосредственно с также OpenGL. Он также включает в себя множество функций, которые дополняют 3D-графики, таких как физики и очень хорошо интегрируется с несколько других платформ Apple, например, Core Animation, Core образа и Sprite Kit.

Дополнительные сведения см. в разделе наших [SceneKit](~/ios/platform/gaming/scenekit.md) документации.

<a name="spritekit" />

### <a name="sprite-kit"></a>Sprite Kit

Sprite Kit, двухмерных игр платформы от Apple, имеет ряд интересных функций в iOS 8 и OS X Yosemite. К ним относятся интеграция с Scene Kit, поддержка построителя текстуры, освещения, теней, ограничения, создания карты нормалей и физики усовершенствования. В частности новые функции физики сделать очень легко добавить реалистичные эффекты для игры.

Дополнительные сведения см. в разделе наших [SpriteKit](~/ios/platform/gaming/spritekit.md) документации.
