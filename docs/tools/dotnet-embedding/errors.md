<a name="EM0000" />

### <a name="em0000-unexpected-error---please-fill-a-bug-report-at-httpsgithubcommonoembeddinator-4000issues"></a>EM0000: Непредвиденная ошибка - заполните отчет об ошибках в https://github.com/mono/Embeddinator-4000/issues

Произошла непредвиденная ошибка. Пожалуйста [сообщите о проблеме](https://github.com/mono/Embeddinator-4000/issues) с как можно больше информации, включая:

* Полная сборка журналы, с максимальным уровнем детализации
* Минимальный тестовый случай, воспроизведите ошибку
* Все версии сведения

Чтобы получить сведения о точной версии проще всего использовать **Xamarin Studio** меню **о Xamarin Studio** элемента, **Показать подробности** кнопку и вставьте в версии информация (можно использовать **сведения о копировании** кнопки).

<a name="EM0001" />

### <a name="em0001-could-not-create-output-directory-x"></a>EM0001: Не удалось создать выходной каталог `X`

Имя каталога, определяемое `-o=DIR` не существует и не может быть создан. Возможно, недопустимое имя для файловой системы.

<a name="EM0002" />

### <a name="em0002-option-x-is-not-supported"></a>EM0002: Параметр `X` не поддерживается

Средство не поддерживает параметр `X`. Вполне возможно, что другая версия средства поддерживает его, или что он не применяется в этой среде.

<a name="EM0003" />

### <a name="em0003-the-platform-x-is-not-valid"></a>EM0003: Платформа `X` является недопустимым.

Средство не поддерживает платформу `X`. Вполне возможно, что другая версия средства поддерживает его, или что он не применяется в этой среде.

<a name="EM0004" />

### <a name="em0004-the-target-x-is-not-valid"></a>EM0004: Целевой `X` является недопустимым.

Средство не поддерживает целевой объект `X`. Вполне возможно, что другая версия средства поддерживает его, или что он не применяется в этой среде.

<a name="EM0005" />

### <a name="em0005-the-compilation-target-x-is-not-valid"></a>EM0005: Целевого объекта компиляции `X` является недопустимым.

Средство не поддерживает целевого объекта компиляции `X`. Вполне возможно, что другая версия средства поддерживает его, или что он не применяется в этой среде.

<a name="EM0006" />

### <a name="em0006-could-not-find-the-xcode-location"></a>EM0006: Не удалось найти расположение Xcode.

Средство не удалось найти текущий выбранный расположение Xcode с помощью `xcode-select -p` команды. Убедитесь, что эта команда выполняется успешно и возвращает правильное расположение Xcode.

<a name="EM0007" />

### <a name="em0007-could-not-get-the-sdk-version-for-sdk"></a>EM0007: Не удалось получить версию пакета sdk для «{sdk}».

Средство не удалось получить версию пакета SDK с помощью `xcrun --show-sdk-version --sdk {sdk}` команды. Убедитесь, что эта команда завершается успешно и возвращает версию пакета SDK.

<a name="EM0008" />

### <a name="em0008-the-architecture-arch-is-not-valid-for-platform-valid-architectures-for-platform-are-architectures"></a>EM0008: Архитектура «{arch}» не является допустимым для {платформа}. Допустимые архитектур для {платформы}: «{архитектур}».

Архитектура в сообщении об ошибке не является допустимым для целевой платформы. Убедитесь, что параметр--abi передается является допустимой архитектурой.

<a name="EM0009" />

### <a name="em0009-the-feature-x-is-not-currently-implemented-by-the-generator"></a>EM0009: Функцию `X` сейчас не реализованы в генератор

Это известная проблема, и мы планируем исправить в следующем выпуске генератора. Вклад приветствуются.

<a name="EM0010" />

### <a name="em0010-cant-merge-the-frameworks-simulatorframework-and-deviceframework-because-the-file-file-exists-in-both"></a>EM0010: Не удается выполнить слияние платформы «{simulatorFramework}» и «{deviceFramework}», так как файл «{file}» существует как в.

Средство не может объединить платформах, упомянутых в сообщении об ошибке, так как между ними общего файла.

Это может указывать на ошибку в внедрение .NET; Отправьте отчет об ошибках в [ https://github.com/mono/Embeddinator-4000/issues ](https://github.com/mono/Embeddinator-4000/issues) с тестовым случаем.

<a name="EM0011" />

### <a name="em0011-the-assembly-x-does-not-exist"></a>EM0011: Сборка `X` не существует.

Средство не удалось найти сборку `X` указанные в аргументах.

<a name="EM0012" />

### <a name="em0012-the-assembly-name-x-is-not-unique"></a>EM0012: Имя сборки `X` не является уникальным

Указано более одной сборки имеют имя же, внутренних и не было бы невозможно различить их во время выполнения.

Наиболее вероятной причиной является то, что сборки указано более одного раза на аргументы командной строки. Тем не менее нельзя будет по-прежнему сохраняет файл переименован сборки, это исходное имя и несколько копий сосуществует.

<a name="EM0013" />

### <a name="em0013-cant-find-the-assembly-x-referenced-by-y"></a>EM0013: Не удается найти сборку «X», ссылается на «Y».

Средство не удалось найти сборку «X», ссылается сборка «Y». Убедитесь, что все сборки, на которую указывает ссылка, в том же каталоге, что и сборка для привязки.

<a name="EM0014" />

### <a name="em0014-could-not-find-product-product-minversion-is-required"></a>EM0014: Не удалось найти {product} ({product} {min_version} является обязательным).

Не удалось найти зависимости, упомянутые в сообщении об ошибке в системе.

<a name="EM0015" />

### <a name="em0015-could-not-find-a-valid-version-of-product-found-version-but-at-least-minversion-is-required"></a>EM0015: Не удалось найти действительную версию {product} (обнаруженная {версия}, но по крайней мере {min_version} является обязательным).

Зависимость упомянутые в ошибку, сообщение найдено в системе, но она устарела. Выполните обновление до более новой версии.

<a name="EM0016" />

### <a name="em0016-could-not-create-symlink-file---target-error-number"></a>EM0016: Не удалось создать символьную ссылку «{файл}» -> «{target}»: ошибка {number}

Не удалось создать символьную ссылку, указано в сообщении об ошибке.

<a name="EM0026" />

### <a name="em0026-could-not-parse-the-command-line-argument-a-"></a>Не удалось EM0026 проанализировать аргумент командной строки «A»: *

Синтаксис для параметра командной строки `A` не удалось проанализировать с помощью средства. Вполне вероятно, неправильными, пожалуйста, свяжитесь с документации или справку для правильного синтаксиса.

<a name="EM0099" />

### <a name="em0099-internal-error--please-file-a-bug-report-with-a-test-case-httpsgithubcommonoembeddinator-4000issues"></a>EM0099: Внутренняя ошибка *. Отправьте отчет об ошибках с тестовым случаем (https://github.com/mono/Embeddinator-4000/issues).

Это сообщение об ошибке выводится при сбое Проверка внутренней согласованности в внедрение .NET.

Это указывает на ошибку в внедрение .NET; Отправьте отчет об ошибках в [ https://github.com/mono/Embeddinator-4000/issues ](https://github.com/mono/Embeddinator-4000/issues) с тестовым случаем.

## <a name="em1xxx-code-processing"></a>EM1xxx: Код обработки

<a name="EM1010" />

### <a name="em1010-type-t-is-not-generated-because-x-are-not-supported"></a>EM1010: Введите `T` не создается, поскольку `X` не поддерживаются.

Это **предупреждение** , тип `T` будет игнорироваться (т. е. ничего не будет создаваться) так как оно использует `X`, это функция, которая не поддерживается.

Примечание: Поддерживаемые функции будет развиваться вместе с новыми версиями средства.

<a name="EM1011" />

### <a name="em1011-type-t-is-not-generated-because-it-lacks-marshaling-code-with-a-native-counterpart"></a>EM1011: Введите `T` не возникает, поскольку у него нет маршалинга кода с помощью собственного какой.

Это **предупреждение** , тип `T` будет игнорироваться (т. е. ничего не будет создаваться), так как он предоставляет, что-то из .NET framework, которая требует дополнительного маршалинга.

Примечание: Это то, что возможно получение поддерживается с некоторыми ограничениями в будущей версии инструмента.

<a name="EM1020" />

### <a name="em1020-constructor-c-is-not-generated-because-of-parameter-type-t-is-not-supported"></a>EM1020: Конструктор `C` не создается из-за параметра типа `T` не поддерживается.

Это **предупреждение** , конструктор `C` будет игнорироваться (т. е. ничего не будет создаваться) так как параметр типа `T` не поддерживается.

Должно быть более ранней предупреждение, почему предоставляет дополнительную информацию об введите `T` не поддерживается.

Примечание: Поддерживаемые функции будет развиваться вместе с новыми версиями средства.

<a name="EM1021" />

### <a name="em1021-constructor-c-has-default-values-for-which-no-wrapper-is-generated"></a>EM1021: Конструктор `C` имеет значения по умолчанию, для которых оболочка не создается.

Это **предупреждение** , параметры по умолчанию конструктора `C` создания дополнительного кода. Наиболее распространенной причиной является то, что существующий метод уже имеет такой же сигнатурой. Например, в .net это может быть:

```
public class MyType {
    public MyType () { ... }
    public MyType (int i = 0) { ... }
}
```

В таких случаях только два созданных `init` селекторы будет создан, оба вызова Mono, но оболочка для более поздних версий не будет существовать.

<a name="EM1030" />

### <a name="em1030-method-m-is-not-generated-because-return-type-t-is-not-supported"></a>EM1030: Метод `M` не создается, поскольку тип возвращаемого значения `T` не поддерживается.

Это **предупреждение** , метод `M` будет игнорироваться (т. е. ничего не будет создаваться) так как это тип возвращаемого значения `T` не поддерживается.

Должно быть более ранней предупреждение, почему предоставляет дополнительную информацию об введите `T` не поддерживается.

Примечание: Поддерживаемые функции будет развиваться вместе с новыми версиями средства.

<a name="EM1031" />

### <a name="em1031-method-m-is-not-generated-because-of-parameter-type-t-is-not-supported"></a>EM1031: Метод `M` не создается из-за параметра типа `T` не поддерживается.

Это **предупреждение** , метод `M` будет игнорироваться (т. е. ничего не будет создаваться) так как параметр типа `T` не поддерживается.

Должно быть более ранней предупреждение, почему предоставляет дополнительную информацию об введите `T` не поддерживается.

Примечание: Поддерживаемые функции будет развиваться вместе с новыми версиями средства.

<a name="EM1032" />

### <a name="em1032-method-m-has-default-values-for-which-no-wrapper-is-generated"></a>EM1032: Метод `M` имеет значения по умолчанию, для которых оболочка не создается.

Это **предупреждение** , параметры по умолчанию метода `M` создания дополнительного кода. Наиболее распространенной причиной является то, что существующий метод уже имеет такой же сигнатурой. Например, в .net это может быть:

```
public class MyType {
    public int Increment () { ... }
    public int Increment (int i = 0) { ... }
}
```

В таких случаях только два созданных `increment` селекторы будет создан, оба вызова Mono, но оболочка для более поздних версий не будет существовать.

<a name="EM1033" />

### <a name="em1033-method-m-is-not-generated-because-another-method-exposes-the-operator-with-a-friendly-name"></a>EM1033: Метод `M` не создается, поскольку другой метод предоставляет оператор с понятным именем.

Это **предупреждение** , метод `M` не создается, поскольку другой метод предоставляет оператор с понятным именем. (https://msdn.microsoft.com/library/ms229032(v=vs.110).aspx)

<a name="EM1034" />

### <a name="em1034-extension-method-m-is-not-generated-inside-a-category-because-they-cannot-be-created-on-primitive-type-t-a-normal-static-method-was-generated"></a>EM1034: Метод расширения `M` не создается внутри категории, поскольку они не может быть создан для типа-примитива `T`. Создан обычный статический метод.

Это **предупреждение** , метод расширения для primivite типа (например `System.Int32`) был найден. В Objective-C не можно создать категории на тип-примитив. Вместо этого генератор будет создавать обычный статический метод.

<a name="EM1040" />

### <a name="em1040-property-p-is-not-generated-because-of-parameter-type-t-is-not-supported"></a>EM1040: Свойство `P` не создается из-за параметра типа `T` не поддерживается.

<a name="EM1051" />

### <a name="em1051-element-e-is-generated-instead-as-f-because-its-name-conflicts-with-an-important-objective-c-selector"></a>EM1051: Элемент `E` вместо этого создается как `F` так как его имя конфликтует с селектора важные objective-c.
